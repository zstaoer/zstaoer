    SpringBoot内容协商

    zstaoer 11/23/2023 Javaspringboot

    springboot整合springmvc内容协商，一个接口返回多个数据格式

    # 自定义返回格式

    现在有一个查询用户信息接口 GET: /users接口

    • 实体

      public class User{
	// ......
}
      1
      2
      3
    1. # 返回json

      springmvc默认返回的是json格式，如下

      {
    "id": 1,
    "name": "张三",
    "age": 79
}
      1
      2
      3

    2. # 返回xml格式

      需要在包管理中导入能够处理xml的依赖，以下采用jackson

      <dependency>
	<groupId>com.fasterxml.jackson.dataformat</groupId>
    <artifactId>jackson-dataformat-xml</artifactId>
</dependency>
      1
      2
      3
      4

      导入依赖后需要在实体上加上注解 @JacksonXmlRootElement

      @JacksonXmlRootElement
public class User{
	// ......
}
      1
      2
      3
      4

      重启项目后请求该接口会发现还是返回json格式的数据。这是因为springmvc默认通过请求头参数来区分返回的格式，在请求头中的accept属性的值默认是application/json,如果需要返回xml格式，需要将accept的值修改为application/xml,此时返回的数据就是xml

      <data>
   <name>zs</name>
   <age>19</age>
   <sex>男</sex>
</data>
      1
      2
      3
      4
      5

      以上就完成了一个接口返回json和xml格式的数据，但是每次都更改请求头很麻烦，接下来增加一个配置，使用参数的形式区分返回哪种数据格式 /users?format=xml

      在配置文件中配置以下信息

      # 开启基于参数的内容协商功能
spring.mvc.contentnegotiation.favor-parameter=true
# 注意，这里写的format，参数后面就只能拼接format /users?format=xml
# 如果这里写为type 则参数拼接 /users?type=xml
spring.mvc.contentnegotiation.parameter-name=format
      1
      2
      3
      4
      5

      还可以支持后缀形式 /users.json,/users.xml

      spring.mvc.contentnegotiation.favor-parameter=true
spring.mvc.contentnegotiation.favor-path-extension=true
spring.mvc.pathmatch.useSuffixPattern=true
      1
      2
      3
    3. # 自定义返回格式

      除了以上两种格式以外，还可以自定义返回格式，比如 text,html,yaml等

      自定义返回格式是基于HttpMessageConverter来实现的,只有在controll上标注@ResponseBody注解或者使用@RestController注解才会生效

      • 在配置文件中添加以下配置

        spring.mvc.contentnegotiation.media-types.yaml
        1

      • 添加能处理yaml文件的依赖，我这里还是使用jackson

        <dependency>
  <groupId>com.fasterxml.jackson.dataformat</groupId>
  <artifactId>jackson-dataformat-yaml</artifactId>
</dependency>
        1
        2
        3
        4

      • 自定义消息转换器

        public class MyYamlHttpMessageConverter extends AbstractHttpMessageConverter<Object> {

    private ObjectMapper objectMapper = null;

    public MyYamlHttpMessageConverter() {
        // 告数springboot这个MessageConverter支持哪种媒体类型
        super(new MediaType("text", "yaml", StandardCharsets.UTF_8));
        // 禁用yaml的环境分割符号 ---
        YAMLFactory factory = new YAMLFactory().disable(YAMLGenerator.Feature.WRITE_DOC_START_MARKER);
        this.objectMapper = new ObjectMapper(factory);
    }

    @Override
    protected boolean supports(Class clazz) {
        // 只要是对象类型，不是基本类型我就放行
        return true;
    }

    @Override
    protected Object readInternal(Class clazz, HttpInputMessage inputMessage) throws IOException, HttpMessageNotReadableException {
        return null;
    }

    @Override
    protected void writeInternal(Object methodReturnValue, HttpOutputMessage outputMessage) throws IOException, HttpMessageNotWritableException {
        try (OutputStream body = outputMessage.getBody()) {
            this.objectMapper.writeValue(body,methodReturnValue);
        }
    }
}
